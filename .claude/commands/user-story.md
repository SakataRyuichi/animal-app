# `/user-story` コマンド

ユーザーストーリーを参照して機能を実装します。

**注意**: このコマンドは`USER_STORIES.md`を参照し、ユーザーの体験価値を重視した実装を行います。

## ワークフロー

1. **ユーザーストーリーの確認**
   - `USER_STORIES.md`から該当するユーザーストーリーを確認
   - 「受け入れ基準」と「体験価値」を理解

2. **実装計画の作成**
   - ユーザーストーリーに基づいて実装計画を作成
   - ユーザーの体験価値を重視した設計

3. **実装の実行**
   - 計画に基づいて実装を実行
   - ユーザーの体験価値を重視したUI/UXを実現

4. **検証**
   - 受け入れ基準を満たしているか確認
   - 体験価値が実現されているか確認

## 使用例

```
/user-story US-009
```

特定のユーザーストーリーを参照して実装：

```
ユーザーストーリードキュメントの US-009（トイレ記録）を実装して。
特に『受け入れ基準』にある視覚的なアイコン選択と、異常時のAI相談への導線を忘れずに。
『体験価値』にある『専門知識がなくても記録できる』UIを重視して作成して。
```

## 注意事項

- ユーザーストーリーは「開発の憲法」として機能します
- 単なる「データ入力フォーム」ではなく、ユーザーの体験価値が込められた「価値ある機能」を作り出す
- キラー機能（AI相談）とフック機能（コラム）の連動を意識
- Cursorのコマンドと併用する場合は、Cursorのコマンドを優先
