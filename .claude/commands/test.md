# `/test` コマンド

テストを実行し、失敗があれば修正します。

**注意**: このコマンドはCursorの`.cursor/commands/test.md`と併用可能です。

## ワークフロー

1. **テストの実行**
   - `pnpm test`を実行して全パッケージのテストを実行
   - または`pnpm --filter <package> test`で特定パッケージのみ
   - または`pnpm maestro test`でMaestro E2Eテストを実行

2. **失敗の確認**
   - テストの失敗を確認
   - 失敗の原因を特定

3. **修正**
   - テストの失敗を修正
   - 必要に応じてテストケースを追加または修正

4. **再実行**
   - 修正後、再度テストを実行
   - すべてのテストが成功するまで繰り返す

5. **完了確認**
   - すべてのテストが成功したことを確認

## 使用例

```
/test
```

特定のパッケージのテストのみ実行：

```
/test --filter expo
```

E2Eテストを実行：

```
/test --e2e
```

## 注意事項

- テストの失敗を無視しない
- テストケースが必要な場合は追加
- 関数型プログラミングの原則に従ってテストを書く
- Cursorのコマンドと併用する場合は、Cursorのコマンドを優先
