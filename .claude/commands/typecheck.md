# `/typecheck` コマンド

TypeScriptの型チェックを実行し、エラーがあれば修正します。

**注意**: このコマンドはCursorの`.cursor/commands/typecheck.md`と併用可能です。

## ワークフロー

1. **型チェックの実行**
   - `pnpm typecheck`を実行して全パッケージの型チェック
   - または`pnpm --filter <package> typecheck`で特定パッケージのみ

2. **エラーの確認**
   - TypeScriptの型エラーを確認
   - エラーの原因を特定

3. **修正**
   - 型エラーを修正
   - 必要に応じて型定義を追加または修正

4. **再実行**
   - 修正後、再度型チェックを実行
   - すべてのエラーが解消されるまで繰り返す

5. **完了確認**
   - すべての型エラーが解消されたことを確認

## 使用例

```
/typecheck
```

## 型エラーの修正方針

- **明示的な型定義**: `any`は避け、適切な型を定義
- **型推論の活用**: 可能な場合は型推論を活用（過度に明示しない）
- **型ガード**: ランタイムチェックが必要な場合は型ガードを使用
- **型アサーション**: 必要な場合のみ使用し、理由をコメントで明記

## 注意事項

- 型エラーを`as any`で無視しない
- 型定義ファイル（`.d.ts`）が必要な場合は作成
- Convexのスキーマ定義（`CONVEX_SCHEMA.md`）を参照して型を確認
- `packages/backend/convex/schema.ts`と`packages/backend/convex/_generated/api.d.ts`を常に参照
- Cursorのコマンドと併用する場合は、Cursorのコマンドを優先
