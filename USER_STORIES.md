# ユーザーストーリー

## 概要
このドキュメントは、アプリの機能をユーザー視点で整理したユーザーストーリーです。
各ストーリーは「As a / I want to / So that」の形式で記述され、ユーザーの感情や体験価値を明確にしています。

**このドキュメントは「開発の憲法」として機能します。**
CursorなどのAI開発ツールに機能の実装を依頼する際、このドキュメントを参照することで、単なる「データ入力フォーム」ではなく、ユーザーの体験価値が込められた「価値ある機能」を作り出すことができます。

### AI（Cursor）への指示への活用

このドキュメントをCursorで活用する際は、以下のように指示してください：

**例1: 基本的な実装依頼**
```
ユーザーストーリードキュメントの US-009（トイレ記録）を実装して。
特に『受け入れ基準』にある視覚的なアイコン選択と、異常時のAI相談への導線を忘れずに。
『体験価値』にある『専門知識がなくても記録できる』UIを重視して作成して。
```

**例2: キラー機能とフック機能の連動**
```
US-027（コラム詳細表示）を実装して。
特に、コラムの末尾にそのトピックに関連したAIへの質問ボタンを追加して。
コラムを読んで「あ、うちの子もこれに当てはまるかも？」と気づいた時、
すぐにAI相談に繋げられるようにして。
```

**例3: 体験価値を重視した実装**
```
US-042（いいね機能）を実装して。
特に、専門家（獣医師など）が「いいね」した投稿には特別なバッジを表示して。
『体験価値』にある『信頼できる情報を発見できる』を実現するUIを重視して。
```

このように指示することで、AIは単なる機能実装ではなく、ユーザーの体験価値を重視した実装を行います。

### ストーリーの構成
各ユーザーストーリーには以下の要素が含まれます：
- **As a**: ユーザーのペルソナ（誰が）
- **I want to**: ユーザーの行動（何をしたいか）
- **So that**: ユーザーの価値・感情（なぜそれが価値があるか）
- **受け入れ基準**: 機能の具体的な要件
- **体験価値**: ユーザーが得られる価値や感情
- **使用シーン**: 実際の使用場面の例
- **優先度**: 開発の優先順位

### フェーズ別の分類
- **Phase 1**: 個人管理・基礎構築（必須機能）
- **Phase 2**: 家族・チーム管理（高優先度）
- **Phase 3**: SNS・商品データベース（中〜高優先度）

### キラー機能・フック機能
- ✅ **キラー機能**: AI相談機能（US-020〜US-025）- アプリの差別化要因
- ✅ **フック機能**: コラム・記事機能（US-026〜US-032）- 毎日アプリを開く動機

**キラー機能とフック機能の連動**:
- **コラム（フック）**: 正しい知識を学び、「あ、うちの子もこれに当てはまるかも？」と気づかせる
- **AI相談（キラー）**: その気づきを即座に専門的なアドバイスへ繋げ、安心感を与える
- **連動の実現**: コラムの末尾に、そのトピックに関連したAIへの質問ボタンを配置（US-027）
  - 例：熱中症のコラムの最後に「うちの子の今の体調を相談する」ボタンを表示
  - これにより、コラムを読んで気づいたことを、すぐにAI相談に繋げられる

## Phase 1: 個人管理・基礎構築

### 認証・ユーザー管理

#### US-001: ユーザー登録
**As a** 初めてペットを飼い始めた飼い主  
**I want to** 簡単にアカウントを作成する  
**So that** すぐにペットの健康管理を始められ、将来の健康トラブルを防げる安心感を得られる

**受け入れ基準**:
- メールアドレスまたはGoogle/Appleアカウントでワンクリック登録できる
- 複雑なパスワード設定やメール認証の待ち時間がない
- 登録後、すぐにペット登録画面に誘導され、スムーズに使い始められる
- プライバシーポリシーと利用規約が分かりやすく表示される

**体験価値**:
- **安心感**: ペットの健康管理を始める第一歩を簡単に踏み出せる
- **時間の節約**: 面倒な登録作業に時間を取られず、すぐに記録を始められる

**優先度**: 最高

#### US-002: ログイン
**As a** 日常的にペットの記録を続けている飼い主  
**I want to** 素早くアプリにアクセスする  
**So that** 毎日の記録を忘れずに行え、ペットの健康状態を常に把握できる安心感を保てる

**受け入れ基準**:
- 生体認証（Face ID/Touch ID）またはワンタップでログインできる
- ログイン後、最後に記録したペットの情報がすぐに表示される
- セッションが維持され、頻繁な再ログインが不要
- オフライン時も最後の状態が表示され、記録の継続性が保たれる

**体験価値**:
- **継続性**: 面倒なログイン作業に阻害されず、毎日の記録習慣を維持できる
- **安心感**: いつでもすぐにペットの情報にアクセスできる安心感

**優先度**: 最高

#### US-003: プロフィール編集
**As a** アプリを使い続けているユーザー  
**I want to** 自分のプロフィール情報を簡単に編集する  
**So that** 最新の情報を反映し、地域に基づいたサービス（近くの病院案内など）を適切に受けられる

**体験価値**:
- **パーソナライズ**: 地域情報を設定することで、より関連性の高いサービスを受けられる
- **継続性**: プロフィールを更新することで、アプリを使い続ける理由が生まれる

**優先度**: 高

### ペット管理

#### US-004: ペット登録
**As a** 新しい家族（ペット）を迎えた飼い主  
**I want to** ペットの基本情報を簡単に登録する  
**So that** ペットの成長記録を残し始められ、将来的に健康管理の履歴として活用できる大切な記録の第一歩を刻める

**受け入れ基準**:
- ペット名、種別、品種、性別、誕生日を直感的なUIで入力できる
- 珍しい種別（爬虫類、小動物など）も「その他」から自由入力でき、すべてのペットが等しく管理できる
- プロフィール画像を撮影または選択でき、ペットの個性が一目で分かる
- 登録後、すぐに記録を始められるよう、記録画面への導線が表示される
- 登録完了時に「これから一緒に健康管理をしていきましょう」という温かいメッセージが表示される

**体験価値**:
- **記念性**: ペットとの生活の始まりを記録する特別な瞬間を大切にできる
- **包括性**: どんな種類のペットでも、同じように大切に管理できる安心感
- **継続性**: 登録後すぐに記録を始められることで、習慣化への第一歩を踏み出せる

**使用シーン**:
- ペットショップから新しい家族を迎えた日
- 保護施設から引き取った日
- 多頭飼いで新しいペットを追加する時

**優先度**: 最高

### 活動ログ記録

#### US-008: 食事記録
**As a** 毎日ペットに食事を与えている飼い主  
**I want to** 食事内容を簡単に記録する  
**So that** 食欲の変化や食事量の推移を把握し、健康状態の早期発見につなげられる。また、かかりつけの獣医師に相談する際に正確な情報を提供できる

**受け入れ基準**:
- 3タップ以内で記録完了できる（種別・量・商品名の入力が簡単）
- よく使う商品は履歴から選択でき、入力の手間が最小限
- 写真を添付して視覚的な記録も残せる（食欲不振時の比較に有効）
- メモで「いつもより食べが悪い」「完食した」などの観察を記録できる
- 記録後、タイムラインに表示され、過去の記録と比較しやすい
- 食事時間のリマインダーを設定でき、記録忘れを防げる

**体験価値**:
- **健康管理**: 食欲の変化を数値で把握でき、病気の早期発見につながる
- **安心感**: 獣医師に相談する際、正確な情報を提供でき、適切な診断を受けられる
- **継続性**: 簡単な操作で毎日の記録が習慣化できる

**使用シーン**:
- 朝・夕の食事タイムにスマホでサッと記録
- 食欲が落ちた時に過去の記録と比較
- 新しいフードに切り替えた時の記録

**優先度**: 最高

#### US-009: トイレ記録
**As a** ペットの健康状態を日々気にかけている飼い主  
**I want to** トイレの状態を簡単に記録する  
**So that** 便の状態や頻度の変化を把握し、消化器系の病気や脱水症状などの健康トラブルを早期に発見できる。特に高齢ペットや持病があるペットの健康管理に不可欠な情報を蓄積できる

**受け入れ基準**:
- トイレタイプ（おしっこ/うんち）をワンタップで選択できる
- 状態（硬い/柔らかい/下痢/血便など）を視覚的なアイコンで選択でき、専門知識がなくても記録できる
- 写真を添付して客観的な記録を残せる（獣医師への相談時に有効）
- メモで「いつもより多い」「色がおかしい」などの観察を記録できる
- 記録後、タイムラインに表示され、過去の状態と比較しやすい
- 異常な状態（下痢、血便など）を記録した場合、AI相談への導線が表示される

**体験価値**:
- **早期発見**: 便の状態の変化を数値化し、病気のサインを見逃さない
- **安心感**: 健康な状態の記録を蓄積することで、異常時の判断材料になる
- **専門家との連携**: 獣医師に相談する際、正確な情報を提供でき、適切な診断を受けられる

**使用シーン**:
- 散歩後のトイレタイムに記録
- 下痢が続いた時の記録とAI相談
- 高齢ペットの健康管理の一環として

**優先度**: 最高

#### US-013: タイムライン表示
**As a** ペットとの日々の記録を大切にしている飼い主

**So that** 過去の記録を振り返り、成長の軌跡を実感でき、健康状態の変化を視覚的に把握できる。また、思い出として楽しめる

**受け入れ基準**:
- 活動ログが時系列で美しく表示され、一目で過去の記録が把握できる
- ログタイプごとに色分けされたアイコンで区別され、視認性が高い
- 日付ごとにグループ化され、特定の日の記録を素早く見つけられる
- 無限スクロールで過去の記録をスムーズに読み込める
- 各ログをタップで詳細表示でき、写真やメモをじっくり見られる
- カレンダーから特定の日付にジャンプでき、過去の記録を効率的に検索できる
- 記録のない日が一目で分かり、記録の継続性を意識できる

**体験価値**:
- **思い出**: ペットとの日々の記録がタイムラインとして残り、成長の軌跡を振り返れる
- **健康管理**: 過去の記録と比較することで、健康状態の変化を視覚的に把握できる
- **継続の動機**: 記録が積み重なることで、継続的な記録へのモチベーションが高まる

**使用シーン**:
- ペットの成長を振り返る時
- 健康状態の変化を確認する時
- かかりつけの獣医師に相談する前に記録を確認

**優先度**: 最高

### ダッシュボード・統計

#### US-016: 今日のサマリー表示
**As a** 毎日の記録を習慣化したい飼い主  
**I want to** ホーム画面で今日の活動サマリーを一目で見る  
**So that** 今日何を記録したか、何がまだ記録されていないかがすぐに分かり、記録漏れを防げる。また、毎日の記録が視覚的に分かることで、記録を続けるモチベーションが高まる

**受け入れ基準**:
- ホーム画面の上部に今日のサマリーが美しいカード形式で表示される
- 今日の食事回数が表示され、目標回数と比較できる
- 今日のトイレ回数が表示され、正常な範囲かどうかが分かる
- 今日の散歩回数・時間が表示され、運動量が把握できる
- 未記録の項目がハイライトされ、記録すべき項目が一目で分かる
- 各項目をタップすると記録画面に遷移でき、すぐに記録できる
- 過去の平均と比較して、「今日は散歩が少ない」などの気づきが表示される

**体験価値**:
- **習慣化**: 記録漏れを防ぎ、毎日の記録習慣を維持できる
- **モチベーション**: 記録が視覚的に分かることで、続ける動機が生まれる
- **健康管理**: 今日の記録を一目で把握し、健康状態の変化に気づきやすい

**使用シーン**:
- 朝、アプリを開いて今日の記録を確認する
- 夜、今日の記録を振り返り、記録漏れがないか確認する
- 記録を忘れた時、未記録の項目を見てすぐに記録する

**優先度**: 高

### コラム・記事機能（Phase 1後半 / Phase 2）✅ **信頼性向上・フック機能**

#### US-026: コラム一覧表示
**As a** ペットの健康について正しい情報を求めている飼い主  
**I want to** 専門家が監修した信頼できるコラム・記事の一覧を見る

- すべての共同管理者が記録をリアルタイムで見られ、情報共有がスムーズ
- 記録時に「家族全員に通知しますか？」というオプションがあり、重要な記録を家族に知らせられる
- 記録の編集・削除は記録者本人またはadmin権限のみが可能で、誤操作を防げる

**体験価値**:
- **協力**: 家族全員でペットの記録を協力して行え、負担が分散される
- **透明性**: 誰がいつ何を記録したかが分かり、情報共有がスムーズ
- **安心感**: 一人が記録を忘れても他の家族が補完でき、記録の継続性が保たれる

**使用シーン**:
- お父さんが朝の散歩を記録し、家族全員が確認できる
- お母さんが食事を記録し、お父さんも同じ情報を見られる
- 子供がペットと遊んだ日記を記録し、家族全員が思い出を共有できる

**As a** ユーザー  
**I want to** 公開日記にいいねをする  
**So that** 気に入った投稿に反応でき、信頼できる情報を他のユーザーと共有できる

**受け入れ基準**:
- いいねボタンをタップできる
- いいね数が増える
- いいねの取り消しができる
- **専門家（獣医師など）が「いいね」した投稿には特別なバッジが表示される**（例：「獣医師が推奨」バッジ）

- 専門家の「いいね」は通常の「いいね」とは区別され、信頼性の指標として表示される
- 「専門家が推奨」バッジが付いた投稿は、おすすめフィードで優先的に表示される

**体験価値**:
- **信頼性**: 専門家が「いいね」した投稿を見ることで、信頼できる情報を発見できる
- **コミュニティ**: 信頼できる飼い主の投稿を識別でき、質の高い情報にアクセスできる
- **モチベーション**: 専門家から「いいね」を受けることで、記録を続けるモチベーションが上がる

**使用シーン**:
- 専門家が推奨した投稿を見て、信頼できる情報を参考にする
- 自分の投稿に専門家が「いいね」してくれて、モチベーションが上がる
- 信頼できる飼い主の投稿を探して、質の高い情報を発見する

**優先度**: 高（Phase 3の信頼性向上機能）
