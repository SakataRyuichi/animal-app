### WEB-019: ニュース・更新情報の自動更新 ✅ **2026年追加 - モノレポ運用**
**As a** 管理者  
**I want to** Convexの`news`テーブルを更新すると、公式サイトのニュース一覧が瞬時に更新される  
**So that** ニュースの更新作業が効率的になり、最新情報を素早く公開できる

**受け入れ基準**:
- Convexの`news`テーブルを更新すると、VercelのOn-demand ISR（Incremental Static Regeneration）により、公式サイトのニュース一覧が瞬時に（かつ静的ファイルとして高速に）更新される
- ニュースの追加・編集・削除が即座に反映される
- パフォーマンス：静的ファイルとして高速に配信される

**体験価値**:
- **効率性**: ニュースの更新作業が効率的になり、最新情報を素早く公開できる
- **パフォーマンス**: 静的ファイルとして高速に配信される

**使用シーン**:
- ニュースを追加・編集・削除する時
- アプリのアップデート情報を公開する時

**優先度**: 高

### WEB-020: 法務ドキュメントの一元管理 ✅ **2026年追加 - モノレポ運用**
**As a** 管理者  
**I want to** 法務ドキュメントを一元管理し、アプリと公式サイトで常に同じ内容を表示できる  
**So that** 法務ドキュメントの更新作業が効率的になり、一貫性が保たれる

**受け入れ基準**:
- `packages/policy/`にMarkdownファイルを置き、アプリの「設定 > 規約」と、公式サイトの「フッター > 規約」が常に同じファイルを読み込むようにビルド設定
- 法務ドキュメントの更新が、アプリと公式サイトの両方に即座に反映される
- バージョン管理と改定履歴を保持

**体験価値**:
- **効率性**: 法務ドキュメントの更新作業が効率的になり、一貫性が保たれる
- **一貫性**: アプリと公式サイトで常に同じ内容を表示できる

**使用シーン**:
- 法務ドキュメントを更新する時
- 新しい法務要件に対応する時

**優先度**: 最高（法的要件）
