# 公式サイトユーザーストーリー インデックス

**📚 ドキュメントインデックス**: [DOCUMENTATION_INDEX.md](./DOCUMENTATION_INDEX.md)

## 概要
このドキュメントは、公式サイト（Next.js + Vercel）の機能をユーザー視点で整理したユーザーストーリーのインデックスです。

**関連ドキュメント**:
- [USER_STORIES.md](./USER_STORIES.md): モバイルアプリ側のストーリー
- [ADMIN_USER_STORIES.md](./ADMIN_USER_STORIES.md): 管理画面側のストーリー
- [CONVEX_SCHEMA.md](./CONVEX_SCHEMA.md): スキーマ定義（news, legal_documentsテーブルなど）
- [DESIGN_DOCUMENT.md](./DESIGN_DOCUMENT.md): アプリ設計の詳細

各ストーリーは「As a / I want to / So that」の形式で記述され、SEOとLLMフレンドリーな構成、アプリのダウンロード導線、ニュース更新、法務ドキュメント管理を目的としています。

**このドキュメントは「開発の憲法」として機能します。**
CursorなどのAI開発ツールに機能の実装を依頼する際、このドキュメントを参照することで、SEOとLLMフレンドリーな公式サイトを作り出すことができます。

## 開発の開始

実装を開始する際は、以下の順序で進めることを推奨します：

1. **Phase 1（必須）**: トップページ、法務ドキュメント、SEO最適化を実装
2. **Phase 2（高優先度）**: ニュース機能、ブランド戦略、モノレポ運用機能を実装
3. **Phase 3（将来機能）**: グローバル公開データの閲覧機能を実装

### AIへの指示例

**例1: トップページの実装**
```
WEB-001（トップページ表示）を実装して。
特に、ストーリーテリング型のコンテンツ構成とSEO最適化を重視して。
メインビジュアルには「記録は、愛した証。」のような情緒的なコピーを配置して。
```

**例2: SEO最適化の実装**
```
WEB-013（SEO最適化）を実装して。
適切なメタタグ、OGPタグ、構造化データ（JSON-LD）を設定して。
LLMフレンドリーな構成も忘れずに。
```

**例3: ブランド戦略の実装**
```
WEB-015（ビジュアル・アイデンティティ）を実装して。
「このアプリがある生活がいかに安心で、温かいか」をブランドとして伝えるデザインにして。
スマートフォンの中でアプリを操作する手元と、その奥でリラックスして眠るペットの「ボケ感」を活かした写真を使用して。
```

## 技術的考慮事項

### SEO
- 適切なメタタグ、OGPタグ、構造化データ（JSON-LD）の設定
- Semantic HTMLの使用
- サイトマップの生成

### LLMフレンドリー
- 構造化されたコンテンツの提供
- FAQセクションの構造化
- 明確な情報アーキテクチャ

### パフォーマンス
- Next.jsのSSG/ISRを活用
- 画像の最適化
- コード分割と遅延読み込み

### UX
- モバイルファーストのデザイン
- アクセシビリティの確保
- ダウンロード導線の最適化
