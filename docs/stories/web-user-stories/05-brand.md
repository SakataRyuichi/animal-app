## ブランド戦略・UI/UXガイドライン

### WEB-015: ビジュアル・アイデンティティ（幸福の想起）✅ **2026年追加 - ブランド戦略**
**As a** 公式サイトのデザイナー・開発者  
**I want to** 「清潔感（信頼）」と「体温（幸福）」の共存を実現する  
**So that** アプリが「実用的な道具」なら、公式サイトは「ペットと飼い主が紡ぐ物語のショーケース」として機能する

**受け入れ基準**:
- **メインビジュアル（ヒーローセクション）**:
  - 構図：スマートフォンの中でアプリを操作する手元と、その奥でリラックスして眠るペットの「ボケ感」を活かした写真
  - メッセージ：「記録は、愛した証。」のような、情緒的で短いコピーを配置
- **カラーパレット**:
  - アプリのベースカラー（例：柔らかなベージュやパステルブルー）を使いつつ、ウェブでは**「余白（ホワイトスペース）」**をアプリの1.5倍広く取り、情報の呼吸を助ける
- **タイポグラフィ**:
  - 見出しには少し丸みのある、親しみやすいフォントを採用
  - ただし、信頼性を損なわないよう、本文は可読性の高い明朝体やクリーンなゴシック体を使い分ける

**体験価値**:
- **信頼性**: 清潔感のあるデザインにより、アプリの信頼性が向上
- **温かみ**: 幸福を想起させるデザインにより、アプリの温かみが伝わる
- **ブランド性**: アプリと公式サイトで一貫したブランドアイデンティティを確立

**使用シーン**:
- 公式サイトのデザインを決定する時
- ブランドガイドラインを作成する時

**優先度**: 高

### WEB-016: UIコンポーネントの共通化と拡張 ✅ **2026年追加 - モノレポ活用**
**As a** 公式サイトの開発者  
**I want to** アプリとUIライブラリを共有し、既視感（安心感）を作る  
**So that** ユーザーがアプリをインストールした際の既視感により、安心感が生まれる

**受け入れ基準**:
- **インタラクティブ・デモ**:
  - アプリで使っている「トイレ記録の選択肢」や「動くフレーム」のUIコンポーネントを、ウェブ上で擬似的に操作できるセクションを設ける
  - ユーザーがクリックすると、バッジが光ったりフレームが動いたりする演出をNext.jsで見せることで、DL前の「手触り感」を伝える
  - `packages/ui`のコンポーネントをウェブ用に拡張して使用
- **カードデザインの共通化**:
  - アプリ内の「日記カード」のデザインをウェブの「ニュース」や「ギャラリー」でも流用
  - `packages/ui`のコンポーネントを共有し、アプリと公式サイトで一貫したデザインを実現

**体験価値**:
- **既視感**: アプリをインストールした際の既視感により、安心感が生まれる
- **手触り感**: インタラクティブ・デモにより、DL前の「手触り感」を伝える
- **効率性**: UIコンポーネントの共通化により、開発効率が向上

**使用シーン**:
- 公式サイトのUIコンポーネントを実装する時
- アプリと公式サイトでデザインを統一する時

**優先度**: 高

### WEB-017: ストーリーテリング型のコンテンツ構成 ✅ **2026年追加 - ブランド戦略**
**As a** 公式サイトを訪れた潜在ユーザー  
**I want to** 上から下へスクロールするだけで、不安が解消され、期待感が高まる構成を体験できる  
**So that** アプリの価値を段階的に理解し、ダウンロードに進める

**受け入れ基準**:
- **Empathy（共感）セクション**:
  - 「今日、何回トイレに行ったっけ？」「この子の小さな変化、見逃したくない。」という飼い主の不安に寄り添う
  - 飼い主の悩みを可視化し、共感を生む
- **Solution（解決）セクション**:
  - アプリの機能（種類別記録、リマインダー）を、実際の操作画面アニメーションで提示
  - インタラクティブ・デモを活用し、機能の価値を実感できる
- **Future（未来）セクション**:
  - 「虹の橋」や「アルバム」の機能を見せ、このアプリが一生（あるいは一生を超えて）寄り添う存在であることを提示
  - ペットとの長期的な関係をイメージさせる
- **Social Proof（信頼）セクション**:
  - バッジを付けた誇らしいペットたちのギャラリーや、最新のアップデートニュース
  - アプリの成長とコミュニティの活発さを示す

**体験価値**:
- **共感性**: 飼い主の不安に寄り添うことで、共感が生まれる
- **理解の深さ**: 段階的な構成により、アプリの価値を深く理解できる
- **期待感**: 未来のセクションにより、アプリへの期待感が高まる
- **信頼性**: Social Proofにより、アプリの信頼性が向上

**使用シーン**:
- 公式サイトのトップページをスクロールする時
- アプリの価値を段階的に理解したい時

**優先度**: 高

### WEB-018: FAQセクションの構造化 ✅ **2026年追加 - SEO・LLM最適化**
**As a** 具体的な悩みを抱えている潜在ユーザー  
**I want to** 「猫の多飲多尿を記録するには？」「爬虫類の掃除頻度は？」といった具体的な悩みへの回答を確認できる  
**So that** 自分の悩みに対する解決策を見つけられる

**受け入れ基準**:
- FAQ形式で具体的な悩みへの回答を配置
- FAQPageの構造化データ（JSON-LD）を付与
- SEO最適化：各FAQに適切なメタタグを設定
- LLMフレンドリー：FAQの内容を構造化された形式で記述
- 検索キーワード対策：具体的な悩みをキーワードとして含める

**体験価値**:
- **検索性**: FAQ形式により、検索エンジンから自然流入が期待できる
- **理解の深さ**: 具体的な悩みへの回答により、アプリの価値を深く理解できる
- **推薦性**: AIがFAQを理解し、アプリを推薦しやすくなる

**使用シーン**:
- 具体的な悩みを検索する時
- AIがアプリの情報を検索する時

**優先度**: 高
