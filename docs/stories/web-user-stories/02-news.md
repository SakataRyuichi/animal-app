## ニュース・更新情報

### WEB-003: ニュース・更新情報一覧表示
**As a** アプリの最新情報を知りたいユーザー  
**I want to** アプリのアップデート情報や機能追加のニュースを確認できる  
**So that** アプリの成長と開発の活発さを実感できる

**受け入れ基準**:
- ニュース一覧を時系列で表示（新しい順）
- 各ニュースにカテゴリ（機能追加、バグ修正、お知らせなど）を表示
- ニュースの公開日時を表示
- SEO最適化：ニュースページに適切なメタタグと構造化データを設定
- LLMフレンドリー：ニュースの内容を構造化された形式で記述

**体験価値**:
- **信頼性**: 開発が活発であることを示し、アプリの信頼性が向上
- **最新性**: 最新の情報を素早く確認できる
- **検索性**: ニュースの内容が検索エンジンにインデックスされ、検索流入が期待できる

**使用シーン**:
- アプリの最新情報を確認したい時
- 新機能が追加されたか確認したい時
- 検索エンジンでアプリの更新情報を検索した時

**優先度**: 高

### WEB-004: ニュース詳細ページ表示
**As a** 特定のニュースの詳細を知りたいユーザー  
**I want to** ニュースの詳細内容を確認できる  
**So that** アップデートの内容を詳しく理解できる

**受け入れ基準**:
- ニュースのタイトル、本文、公開日時を表示
- 関連するスクリーンショットや画像を表示
- 前後のニュースへのナビゲーション
- SEO最適化：各ニュースページに適切なメタタグと構造化データを設定
- OGPタグ：SNS共有時に適切なプレビューが表示される

**体験価値**:
- **理解の深さ**: アップデートの詳細を理解できる
- **共有性**: SNSで共有しやすく、拡散が期待できる
- **検索性**: ニュースの内容が検索エンジンにインデックスされ、検索流入が期待できる

**使用シーン**:
- 特定のニュースの詳細を確認したい時
- ニュースをSNSで共有したい時
- 検索エンジンで特定のニュースを検索した時

**優先度**: 高

### WEB-005: ニュース・更新情報の管理（管理者）
**As a** 管理者  
**I want to** ニュースや更新情報を追加・編集・削除できる  
**So that** アプリの最新情報を公式サイトで公開できる

**受け入れ基準**:
- Convexの`news`テーブルにニュースを追加・編集・削除できる
- ニュースのタイトル、本文、カテゴリ、公開日時を設定できる
- 下書き保存機能
- 公開・非公開の切り替え
- Markdown形式での記述に対応

**体験価値**:
- **効率性**: 簡単にニュースを追加・更新できる
- **柔軟性**: 下書き保存や公開制御により、公開タイミングを調整できる

**使用シーン**:
- アプリのアップデート情報を公開する時
- 新機能の追加を告知する時
- お知らせを公開する時

**優先度**: 高
