## コラム・記事機能

#### US-026: コラム一覧表示
**画面パス**: `app/(tabs)/articles.tsx`, `app/articles/index.tsx`
**As a** ペットの健康について正しい情報を求めている飼い主  
**I want to** 専門家が監修した信頼できるコラム・記事の一覧を見る  
**So that** SNSの不確かな情報に惑わされず、科学的根拠に基づいた正しい知識を得られ、ペットの健康を守る判断ができる

**受け入れ基準**:
- 公開済みのコラムが美しいカード形式で一覧表示され、一目で内容が分かる
- 飼っているペットの種別でフィルターでき、関連性の高い情報にすぐアクセスできる
- タグ（「初心者」「住環境」「食事」など）でフィルターでき、知りたい情報を効率的に探せる
- 新しい順・人気順でソートでき、最新情報や多くの人が読んでいる記事を優先的に見られる
- アイキャッチ画像とタイトルが魅力的に表示され、読みたい記事を選びやすい
- 各コラムに「専門家監修」「一次ソースあり」などの信頼性を示すバッジが表示される

**体験価値**:
- **信頼性**: SNSの不確かな情報ではなく、専門家が監修した信頼できる情報を得られる安心感
- **効率性**: 自分が知りたい情報に素早くアクセスでき、時間を無駄にしない
- **学習**: ペットの健康管理に関する正しい知識を継続的に学べる

**使用シーン**:
- 新しいペットを迎えた時、基本的な知識を学ぶ
- ペットの行動に疑問を持った時、正しい情報を探す
- 毎日の習慣として、新しいコラムをチェックする

**優先度**: 高

#### US-027: コラム詳細表示
**画面パス**: `app/articles/[id].tsx`
**As a** ペットの健康について深く学びたい飼い主  
**I want to** コラムの詳細を読みやすい形式で読む  
**So that** 専門家が監修した信頼できる情報を深く理解でき、ペットの健康管理に実践的に活用できる。また、一次ソースを確認することで、さらに詳しく学べる

**受け入れ基準**:
- タイトル、本文（Markdown形式）が読みやすいフォーマットで表示され、長文でも読みやすい
- アイキャッチ画像が美しく表示され、記事の内容が視覚的に伝わる
- 一次ソースのリンクが明確に表示され、タップで確認できる
- 著者情報が表示され（専門家の場合）、「獣医師監修」「専門家執筆」などのバッジが付く
- **コラムの末尾に、そのトピックに関連したAIへの質問ボタンが表示される**（例：熱中症のコラムの最後に「うちの子の今の体調を相談する」ボタン）✅ **キラー機能とフック機能の連動**
- AI相談ボタンをタップすると、コラムの内容をコンテキストとして含めた状態でAI相談画面が開く
- 関連コラムが「この記事を読んだ人は、こちらも読んでいます」としてレコメンドされ、学習が深まる
- 記事の重要ポイントがハイライトされ、忙しい時でも要点を把握できる
- 「後で読む」機能があり、後でじっくり読める

**体験価値**:
- **学習**: 専門家が監修した信頼できる情報を深く理解でき、実践的に活用できる
- **信頼性**: 一次ソースを確認できることで、情報の信頼性を自分で検証できる
- **即時性**: コラムを読んで「あ、うちの子もこれに当てはまるかも？」と気づいた時、すぐにAI相談に繋げられる（キラー機能とフック機能の連動）
- **継続性**: 関連コラムがレコメンドされることで、学習が継続的に深まる

**使用シーン**:
- 新しいペットを迎えた時、基本的な知識を深く学ぶ
- ペットの行動に疑問を持った時、詳しい情報を読む
- コラムを読んで「うちの子もこれに当てはまるかも？」と気づいた時、すぐにAI相談ボタンをタップして相談する
- 熱中症のコラムを読んだ後、「うちの子の今の体調を相談する」ボタンでAI相談を開始する
- 時間がある時、関連コラムを読んで学習を深める

**優先度**: 高

#### US-028: レコメンドコラム表示
**画面パス**: `app/(tabs)/articles.tsx`, `app/articles/index.tsx`
**As a** 毎日アプリを開く習慣を作りたい飼い主  
**I want to** ホーム画面で自分が飼っているペットに合わせたコラムを見る  
**So that** アプリを開くたびに新しい学びや気づきがあり、毎日の記録習慣と情報収集が自然に結びつく。これにより、アプリを開く動機が生まれ、継続的な利用につながる

**受け入れ基準**:
- ホーム画面の上部に「あなたにおすすめ」セクションが目立つように表示される
- 飼っているペットの種別に合わせたコラムが自動的に表示され、自分で探す手間がない
- 新しいコラムが追加されたらプッシュ通知で知らせられ、見逃さない
- コラムのタイトルとアイキャッチが魅力的で、クリックしたくなる
- 読んだコラムは「既読」マークが付き、新しいコラムが一目で分かる
- コラムを読んだ後、関連する記録機能への導線が表示される（例：「このコラムを読んだら、住環境を記録してみませんか？」）

**体験価値**:
- **習慣化**: アプリを開くたびに新しい情報があり、開く動機が生まれる
- **パーソナライズ**: 自分に関連性の高い情報が自動的に表示され、情報収集の効率が上がる
- **継続性**: 記録と情報収集が自然に結びつき、アプリを使い続ける理由になる

**使用シーン**:
- 朝、アプリを開いて今日の記録を確認する時、おすすめコラムを読む
- ペットの行動に疑問を持った時、関連コラムが表示される
- 新しいコラムが追加された通知を受け取り、すぐに読む

**優先度**: 最高（フック機能）

#### US-079: 管理者厳選のキュレーション一覧表示
**画面パス**: `app/curation/index.tsx`
**As a** ペット飼い主  
**I want to** 管理者が厳選した外部記事の一覧を見る  
**So that** 信頼できる情報源から、ペットの健康や飼育に関する最新情報を得られる

**受け入れ基準**:
- キュレーション一覧画面で、管理者が厳選した外部記事が表示される
- 記事のタイトル、管理者による紹介文（summary）、サムネイル画像が表示される
- カテゴリ（健康・医療、食事・栄養、ライフスタイル、ケア・美容、緊急時対応）でフィルタリングできる
- ペットの種別に合わせて、関連する記事が優先的に表示される（パーソナライズ）
- 新しい順にソートされる
- **無料ユーザー**: 記事のタイトルと要約（summary）まで閲覧可能
- **プレミアムユーザー**: 外部記事へのアクセスと、その記事に基づいた「AIアドバイス」の受領が可能
- 記事をタップすると、アプリ内ブラウザで記事が開く（アプリから離脱しない）

**体験価値**:
- **信頼性**: 管理者が厳選した記事なので、信頼できる情報源から情報を得られる
- **パーソナライズ**: 自分のペットの種別や年齢に合わせて、関連する記事が優先的に表示される
- **継続性**: アプリ内ブラウザを使用することで、アプリから離脱せずに情報を得られる

**使用シーン**:
- ペットの健康について最新情報を知りたい時
- 新しいフードやケア方法について調べたい時
- 緊急時の対応方法を確認したい時

**優先度**: 高

#### US-080: キュレーション記事の閲覧
**画面パス**: `app/curation/[id].tsx`
**As a** ペット飼い主  
**I want to** 管理者が厳選した外部記事を読む  
**So that** 信頼できる情報源から、ペットの健康や飼育に関する情報を得られる

**受け入れ基準**:
- 記事をタップすると、アプリ内ブラウザ（`expo-web-browser`）で記事が開く
- 左上の「完了」ボタンで即座にアプリに戻れる
- 記事を開く前に、「この記事のここがレオくんに役立つかも！」というパーソナライズされたポップアップが表示される（AIがユーザーのペット情報を元に生成）
- 記事閲覧中に、「この記事について話す」ボタンが表示され、アプリ内のAIチャットに導線がある
- 「あとで読む」ボタンで、後から読めるように保存できる
- 「この記事を参考にアルバムを作る」ボタンで、外部記事を自分たちのペットの記録と紐付けられる
- 記事の閲覧履歴が記録される（`curation_interactions`テーブル）

**体験価値**:
- **継続性**: アプリ内ブラウザを使用することで、アプリから離脱せずに情報を得られる
- **パーソナライズ**: 自分のペットに合わせた紹介文で、記事の価値が分かりやすい
- **統合性**: 外部記事をアプリ内の記録やアルバムと紐付けられる

**使用シーン**:
- 管理者が厳選した記事を読む時
- 記事を参考に、ペットの記録やアルバムを作る時
- 記事について、AIに相談したい時

**優先度**: 高

#### US-081: キュレーション記事の「あとで読む」機能
**画面パス**: `app/curation/[id].tsx`
**As a** ペット飼い主  
**I want to** 気になる記事を「あとで読む」に保存する  
**So that** 後から時間がある時に、ゆっくり読める

**受け入れ基準**:
- 記事一覧画面や記事詳細画面で「あとで読む」ボタンをタップできる
- 「あとで読む」に追加された記事が一覧で表示される
- 「あとで読む」から記事を削除できる
- 「あとで読む」の記事をタップすると、アプリ内ブラウザで記事が開く

**体験価値**:
- **効率性**: 気になる記事を後から読めるように保存できる
- **整理**: 読みたい記事を一箇所にまとめられる

**使用シーン**:
- 時間がない時に、気になる記事を見つけた時
- 後でゆっくり読むために、記事を保存したい時

**優先度**: 中

#### US-082: キュレーション記事をアルバムに保存
**画面パス**: `app/curation/[id].tsx`
**As a** ペット飼い主  
**I want to** 外部記事を参考にアルバムを作る  
**So that** 記事の内容を自分のペットの記録と紐付けられる

**受け入れ基準**:
- 記事閲覧画面で「この記事を参考にアルバムを作る」ボタンをタップできる
- アルバム作成画面が開き、記事のタイトルとURLが自動的に追加される
- 記事を参考にした日記や写真をアルバムに追加できる
- アルバムの説明に、参考にした記事の情報が含まれる

**体験価値**:
- **統合性**: 外部記事をアプリ内の記録やアルバムと紐付けられる
- **価値の向上**: 記事の内容を自分のペットの記録として残せる

**使用シーン**:
- 記事を参考に、ペットの記録やアルバムを作る時
- 記事の内容を自分のペットの記録として残したい時

**優先度**: 中
