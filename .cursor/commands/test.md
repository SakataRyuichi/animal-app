# `/test` コマンド

コード変更後にテストを実行し、失敗があれば修正します。

## ワークフロー

1. **テストの実行**
   - 変更されたパッケージのテストを特定
   - `pnpm --filter <package> test`を実行
   - または`pnpm test`で全パッケージのテストを実行

2. **失敗の確認**
   - テストが失敗した場合、エラーメッセージを分析
   - 失敗の原因を特定

3. **修正**
   - テストが失敗している原因を修正
   - 必要に応じてテストコードも更新

4. **再実行**
   - 修正後、テストを再実行
   - すべてのテストが通るまで繰り返す

5. **完了確認**
   - すべてのテストが成功したことを確認
   - 変更内容を要約

## 使用例

```
/test
```

または、特定のパッケージのテストのみ実行：

```
/test --filter mobile
```

## 注意事項

- テストがない場合は、新しくテストを作成することを提案
- テストの実行時間が長い場合は、単一のテストファイルを指定して実行
- E2Eテスト（Maestro）は時間がかかるため、必要に応じてのみ実行
