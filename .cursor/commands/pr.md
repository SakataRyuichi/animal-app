# `/pr` コマンド

現在の変更に対してプルリクエストを作成します。

## ワークフロー

1. **変更内容の確認**
   - `git status`で変更されたファイルを確認
   - `git diff`でステージされた変更とステージされていない変更を確認

2. **検証の実行**
   - `/verify`コマンドを実行して、型チェック、リント、テストを実行
   - すべての検証が成功することを確認

3. **コミットメッセージの作成**
   - 変更内容に基づいて明確なコミットメッセージを作成
   - 変更の種類（feature, fix, refactorなど）をプレフィックスに含める

4. **コミットとプッシュ**
   - 変更をステージング (`git add .`)
   - コミット (`git commit -m "..."`)
   - 現在のブランチにプッシュ (`git push`)

5. **プルリクエストの作成**
   - `gh pr create`を使用してプルリクエストを作成
   - タイトルと説明を自動生成
   - 関連するIssueがあればリンク

6. **PR URLの返却**
   - 作成されたPRのURLを返す

## 使用例

```
/pr
```

または、特定の説明を追加：

```
/pr: コラム機能の追加を実装しました
```

## コミットメッセージの形式

```
<type>: <description>

<詳細説明（オプション）>
```

### Typeの種類

- `feat`: 新機能
- `fix`: バグ修正
- `refactor`: リファクタリング
- `docs`: ドキュメント更新
- `test`: テスト追加・修正
- `chore`: その他の変更

## 注意事項

- コミット前に必ず検証を実行
- コミットメッセージは明確で説明的に
- 関連するIssueがあれば、PR説明に含める
