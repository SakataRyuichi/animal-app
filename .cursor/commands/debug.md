# `/debug` コマンド

問題をデバッグし、根本原因を特定して修正します。

## ワークフロー

1. **問題の理解**
   - エラーメッセージを確認
   - 問題が発生している場所を特定
   - 再現手順を確認

2. **ログの追加**
   - 問題の原因を特定するためにログを追加
   - デバッグ情報を出力
   - 実行時の状態を確認

3. **テストの追加（推奨）**
   - 問題を再現するテストを追加
   - テストが失敗することを確認
   - これにより、修正後に問題が解決したことを検証できる

4. **根本原因の特定**
   - ログとエラーメッセージを分析
   - 問題の根本原因を特定

5. **修正**
   - 根本原因を修正
   - 症状だけでなく、根本原因に対処する

6. **検証**
   - テストを実行して問題が解決したことを確認
   - ログを確認して期待通りの動作を確認
   - 型チェックとリントを実行

## 使用例

```
/debug
```

または、特定の問題をデバッグ：

```
/debug: ログイン後にユーザー情報が表示されない問題を調査してください。
```

## デバッグ戦略

### 1. ログベースのデバッグ

```typescript
// デバッグログを追加
console.log("[DEBUG] Current state:", { userId, pets });
```

### 2. テストベースのデバッグ

```typescript
// 問題を再現するテストを追加
test("should handle login error", async () => {
  // テストコード
});
```

### 3. 段階的なデバッグ

- 問題を小さな部分に分割
- 各部分を個別にデバッグ
- 段階的に問題を特定

## 注意事項

- ログは本番環境では削除するか、適切なログレベルを使用
- デバッグログは一時的なものとし、問題解決後は削除
- 根本原因を修正し、症状だけを対処しない
