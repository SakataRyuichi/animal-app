# `/lint` コマンド

ESLintを実行し、エラーがあれば自動で修正します。

## ワークフロー

1. **リントの実行**
   - `pnpm lint`を実行して全パッケージのリントをチェック
   - または`pnpm --filter <package> lint`で特定パッケージのみ

2. **エラーの確認**
   - ESLintのエラーと警告を確認
   - 自動修正可能なエラーを特定

3. **自動修正**
   - `pnpm lint --fix`を実行して自動修正可能なエラーを修正
   - または`pnpm format`でPrettierによるフォーマットを実行

4. **手動修正が必要なエラーの報告**
   - 自動修正できないエラーをリスト化
   - 各エラーに対して修正方法を提案

5. **修正の実行**
   - 手動修正が必要なエラーを修正
   - 修正後、再度リントを実行して確認

## 使用例

```
/lint
```

または、自動修正を含める：

```
/lint --fix
```

## 注意事項

- 自動修正は安全な変更のみ（フォーマット、不要なインポート削除など）
- ロジックの変更が必要なエラーは、手動で修正を提案
- リントエラーを無視する場合は、適切なESLintコメントを使用（理由を明記）
