# `/verify` コマンド

開発後の自動検証フローを実行します。テスト、リント、型チェック、セキュリティ検証を順番に実行し、すべてが成功することを確認します。

## ワークフロー

1. **型チェック**
   - `pnpm typecheck`を実行
   - エラーがあれば修正してから次へ

2. **リント**
   - `pnpm lint`を実行
   - 自動修正可能なエラーを修正
   - 手動修正が必要なエラーを報告

3. **フォーマット**
   - `pnpm format`を実行してコードをフォーマット

4. **テスト**
   - `pnpm test`を実行
   - 失敗があれば修正

5. **セキュリティ検証** ⭐ **2026年追加**
   - 包括的なセキュリティ検証を実行
   - セキュリティ監査、依存関係監査、Terraformセキュリティスキャン

6. **結果の要約**
   - すべての検証が成功したことを確認
   - 修正した内容を要約

## 使用例

```
/verify
```

## 検証項目

- ✅ TypeScriptの型チェック
- ✅ ESLintによるコード品質チェック
- ✅ Prettierによるフォーマット
- ✅ テストの実行
- ✅ セキュリティ検証（IPAガイドライン準拠） ⭐ **2026年追加**

## 注意事項

- 各ステップでエラーが発生した場合は、修正してから次のステップに進む
- すべての検証が成功するまで繰り返す
- 検証に時間がかかる場合は、進捗を報告

## セキュリティ検証の詳細

セキュリティ検証では、以下のチェックを実行します：

1. **セキュリティ監査**: IPAガイドラインに基づくセキュリティチェック
2. **依存関係監査**: 依存ライブラリの脆弱性スキャン
3. **Terraformセキュリティスキャン**: Cloudflare R2バケットの設定確認

詳細は [.cursor/commands/security-verify.md](./security-verify.md) を参照してください。
