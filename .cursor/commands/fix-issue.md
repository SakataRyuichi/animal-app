# `/fix-issue` コマンド

GitHub Issueを分析し、関連するコードを見つけて修正を加え、PRを作成します。

## ワークフロー

1. **Issueの取得**
   - `gh issue view <number>`でIssueの詳細を取得
   - Issueの説明、ラベル、コメントを確認

2. **関連コードの探索**
   - Issueの内容に基づいて関連するファイルを検索
   - 問題が発生している箇所を特定

3. **問題の理解**
   - Issueの内容を理解
   - 再現手順を確認
   - 期待される動作を確認

4. **修正の実装**
   - 問題を修正
   - 必要に応じてテストを追加
   - 検証を実行 (`/verify`)

5. **PRの作成**
   - 変更をコミット
   - プルリクエストを作成
   - Issueへのリンクを含める

## 使用例

```
/fix-issue 123
```

## 注意事項

- Issueの内容を正確に理解してから修正を開始
- 修正後は必ずテストを追加または実行
- PR説明にIssue番号を含める（`Fixes #123`）
