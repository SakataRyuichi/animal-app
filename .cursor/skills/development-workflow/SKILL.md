# 開発ワークフロースキル

コード変更後の自動検証とデバッグを支援するスキルです。

## 自動検証フロー

コード変更後は、以下の順序で検証を実行してください：

1. **型チェック** (`pnpm typecheck`)
   - TypeScriptの型エラーがないか確認
   - エラーがあれば修正

2. **リント** (`pnpm lint`)
   - ESLintのエラーと警告を確認
   - 自動修正可能なエラーを修正

3. **フォーマット** (`pnpm format`)
   - Prettierでコードをフォーマット

4. **テスト** (`pnpm test`)
   - ユニットテストを実行
   - 失敗があれば修正

## デバッグフロー

問題が発生した場合のデバッグ手順：

1. **問題の再現**
   - 問題を再現する手順を確認
   - エラーメッセージを確認

2. **ログの追加**
   - 問題の原因を特定するためにログを追加
   - デバッグ情報を出力

3. **テストの追加**
   - 問題を再現するテストを追加
   - テストが失敗することを確認

4. **修正**
   - 問題の根本原因を特定
   - 修正を実装

5. **検証**
   - テストが成功することを確認
   - 問題が解決したことを確認

## 使用例

コード変更後、自動検証を実行：

```
コードを変更しました。自動検証フローを実行してください。
```

問題が発生した場合：

```
この問題をデバッグしてください。まず、問題を再現するテストを追加してから修正してください。
```
